05  LLM in practice

LLM을 써본 사람이 아니라,
LLM을 다뤄본 사람

GEO의 대상이 LLM인 만큼, 모델이 무엇에 어떻게 반응하는지를 직접 다뤄본 경험이 이 일의 출발점이라고 생각합니다.

같은 입력에도 모델은 다르게 반응합니다.

저는 여러 LLM을 하나의 파이프라인으로 묶어 단독으로 만들고 운영하며, Claude·GPT·Gemini가 같은 입력에 다르게 반응하는 패턴, 어떤 맥락과 표현이 출력을 바꾸는지를 반복해 다뤘습니다. 이는 LLM이 어떤 콘텐츠를 어떤 조건에서 인용·노출하는지를 진단하는 GEO 작업의 토대입니다.

또한 자동 수집·분석 파이프라인을 API 기반으로 직접 구현해, 모델 응답과 데이터를 모으고 가공하는 분석 환경을 만들 수 있습니다. 분석을 도구에 의존하지 않고 필요한 측정 환경을 직접 짤 수 있다는 것이 제 강점입니다.

각 설계 판단은 Context · Decision · Trade-off 구조로 정리해, 왜 그렇게 만들었는지 설명할 수 있도록 남겨 두었습니다.
